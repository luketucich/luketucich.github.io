Project Overview

UNCG Sewing Club Website - Requirements & Design

Project Overview

  1. Describe your application and its purpose: The UNCG Sewing Club Website is a web application designed to promote the UNCG Sewing Club, founded by Bertha Jernigan. Its purpose is to showcase club activities, provide information on how to join, highlight past events, and share details on upcoming meetings to encourage participation and communication among members and interested students. The site will use a cute, pastel-colored theme with pink, light blue, and mint tones for a girly, welcoming aesthetic.
  2. Intended users of the website: Current Sewing Club members (for updates and event recaps) and potential members (UNCG students interested in sewing, crafting, or community activities).
  3. Overview of the content of the website: Content includes club introduction and goals, schedules for meetings, photo galleries from events, contact forms for inquiries or joining, and details on past meetings with photos and summaries. No user data storage; all content is static or loaded read-only via APIs if needed.

Dynamic Functionality on the Website

  • Contact Form with Validation (JavaScript): On the Contact page, a form for inquiries with client-side checks (e.g., empty fields, valid email). Purpose: Collect info safely without server storage. Example URL: https://www.w3schools.com/js/js_validation.asp (demonstrates basic JS form validation).
  • jQuery-UI Tooltip: On Gallery images and links across pages, hover to show extra info like event dates or descriptions. Purpose: Provide quick context without cluttering the page. Example URL: https://jqueryui.com/tooltip/ (official demo of jQuery-UI tooltips).
  • jQuery-UI Accordion: On the Past Meetings page, collapsible sections for each event to expand details. Purpose: Organize content for better readability on mobile/desktop. Example URL: https://jqueryui.com/accordion/ (official demo of jQuery-UI accordion).
  • AJAX Functionality: On the Schedule page, load upcoming events from a JSON file without refreshing. Purpose: Dynamic updates if events change, keeping the site fresh. Example URL: https://api.jquery.com/jQuery.getJSON/ (shows how to load JSON via AJAX).
